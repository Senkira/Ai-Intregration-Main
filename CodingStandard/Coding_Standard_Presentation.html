Project Structure โครงสร้างโปรเจค — 10 เกณฑ์ (7 บังคับ + 3 แนะนำ)
📖 อ้างอิง: Microsoft N-Layer Architecture — ออกแบบสำหรับ Single-Project Monolith API + Dapper
#เกณฑ์ระดับ
1.1แยกโฟลเดอร์ 3 Layer อย่างน้อย: Controllers/, Services/, DataAccess/บังคับ
1.2Models/ ต้องแยก — มี Entities/ สำหรับ DB Mapping และ Requests/ สำหรับ Input DTOบังคับ
1.3Services/Interfaces/ สำหรับ Service Interface (SOLID — Dependency Inversion)บังคับ
1.4DataAccess/ ต้องมี Connections/ + Repositories/บังคับ
1.5Repositories/Interfaces/ สำหรับ Repository Interface — Mock ได้ใน Unit Testบังคับ
1.6Filters/ แยก Action Filter — ห้ามวางใน Controllerบังคับ
1.7tests/ แยกจาก src/ — Production build ไม่ compile testบังคับ
1.8.editorconfig ที่ Solution Root — บังคับ Code Style ร่วมกันแนะนำ
1.9SqlQueries/ แยก SQL Constants ตาม Feature — ใช้เมื่อ Query มากแนะนำ
1.10Middleware/ สำหรับ Custom Middleware — สร้างเมื่อจำเป็นแนะนำ

DataAccess Layer Dapper + Repository Pattern — 38 เกณฑ์ (ทั้งหมดบังคับ)
⚠️ Dapper เป็น Micro-ORM เดียวที่อนุญาต — ห้ามใช้ EF Core, NHibernate หรือ ORM อื่น · Performance ≤5% gap จาก ADO.NET · Code กระชับกว่า 5-10 เท่า

📦 Data Access Pattern

#เกณฑ์
3.1ใช้ Dapper เท่านั้น — ห้ามใช้ ORM อื่น (EF Core, NHibernate)
3.2ใช้ Repository Pattern — ห้าม inject IDbConnection เข้า Service ตรง
3.3มี IDbConnectionFactory — ห้าม new SqlConnection() ตรงใน Repository
3.4มี BaseRepository เป็น Generic Base Class wrap Dapper methods

🛡️ SQL Security & Management

#เกณฑ์Security
3.6ห้าม String Interpolation ใน SQL — ต้องใช้ @parameter เสมอCWE-89
3.10Connection ต้องปิดด้วย using เสมอCWE-404
3.11ห้าม SELECT * — ระบุ Column ที่ต้องการเท่านั้นCWE-200
3.15ห้าม .Result / .Wait() — ต้อง await เสมอCWE-833
3.21Connection String ห้ามอยู่ใน Source Code — ใช้ appsettings/VaultCWE-798
3.23Encrypt=True; TrustServerCertificate=False สำหรับ ProductionCWE-311
3.26Database Account = Least Privilege — ห้ามใช้ sa / db_ownerCWE-250

🔄 Async & Resilience

#เกณฑ์
3.12ทุก Repository method = async Task<T>
3.13ทุก async ต้องรับ CancellationToken เป็น parameter สุดท้าย
3.14ส่ง CT ผ่าน CommandDefinition (Dapper ไม่รับ CT โดยตรง)
3.18Retry Policy (Polly) — Exponential Backoff 1s→2s→4s, max 3 ครั้ง
3.27ทุก INSERT/UPDATE/DELETE ต้องมี Audit Log — Who/What/When

Service Layer ชั้น Business Logic — 8 เกณฑ์ (ทั้งหมดบังคับ)
#เกณฑ์ถ้าไม่ทำ → เกิดอะไร
4.1ทุก Service ต้องมี Interface ใน Services/Interfaces/DI/Mock ไม่ได้ → test ไม่ได้
4.2Constructor Injection เท่านั้น — ห้าม new dependencyCoupling แน่น → test ยาก
4.3ใช้ IRepository เข้าถึง DB — ห้ามใช้ IDbConnection ตรงSQL กระจาย → maintain ลำบาก
4.4async + CancellationToken ทุก I/O method → ส่งต่อถึง RepositoryThread block → server รับ load ไม่ได้
4.5Guard Clause (null/empty check) ต้นเมธอด ก่อน Business LogicNullReferenceException crash
4.6try-catch + Structured Logging — return ResultModel, ห้าม leak exceptionError หลุดถึง client = security risk
4.7Return Response DTO — ห้าม return Entity ตรงClient เห็น DB column ทั้งหมด = data leak
4.8ห้าม Hard-coded value — ค่า config มาจาก IOptions<T>Deploy ต้องแก้ code + build ใหม่
💡 Defense-in-Depth: Guard Clause (§4.5) = defensive check ใน Service · Primary Validation = FluentValidation ที่ Controller Level (§15) · ทั้งสองทำงานร่วมกันตาม OWASP Input Validation Cheat Sheet

Controller Layer ชั้น API Endpoint — 8 เกณฑ์ (ทั้งหมดบังคับ)
#เกณฑ์ถ้าไม่ทำ → เกิดอะไร
5.1สืบทอดจาก ControllerBase (ไม่ใช่ Controller)ได้ View support ที่ไม่จำเป็น
5.2Attribute: [ApiController], [Route], [Produces("application/json")]ไม่ได้ auto model validation
5.3รับ IService ผ่าน Constructor — ห้ามมี Business LogicTest ยาก + Reuse ไม่ได้
5.4ทุก Endpoint ที่ต้อง Auth ต้องมี [Authorize]Endpoint เปิด public = security hole
5.5ระบุ HTTP Verb: [HttpGet], [HttpPost], [HttpPut], [HttpDelete]Swagger doc ไม่ถูก + 405 error
5.6มี HealthCheckController อย่างน้อย 1 endpoint ไม่ต้อง AuthK8s probe ไม่ได้ → pod restart ผิด
5.7Controller = รับ → เรียก → ส่ง (1-3 บรรทัดต่อ action)Business logic ปน → SRP ผิด
5.8ทุก async Action รับ CancellationToken ส่งต่อให้ ServiceClient disconnect → query ยังวิ่ง
🔗 CancellationToken Chain: Controller (§5) → Service (§4) → Repository (§3) — ASP.NET Core bind HttpContext.RequestAborted อัตโนมัติ
